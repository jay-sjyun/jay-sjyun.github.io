OVERVIEW

프로젝트 소개
유니티 엔진으로 제작된 기존의 ‘캐어유 정신건강테스트 앱’을 유지/보수에 용이하도록 리액트 네이티브 언어로 재구축하였습니다. 동시에, 기존 디자인이 지닌 문제점 파악과 사용자 피드백을 바탕으로 UI/UX를 개선하여 앱을 리디자인하였습니다.

2021년 7월 ~ 8월
참여인원

프론트엔드 개발자 1명
백엔드 개발자 1명
역할
UI/UX 디자인 전담 / 퍼블리싱 전담

PERSONA

이영숙

  • 만 67세
  • 은퇴 후 여가생활
  • 건강에 관심
  • 치매 가족력

목표

  • 주기적인 정신건강 상태 체크
  • 간단한 검사로 치매 증상 조기 발견
  • 정신건강 상태 추이를 수치로 확인

불만사항

  • 스마트 기기 숙련도가 낮아 복잡한 UI 학습이 스트레스로 다가옴
  • 근시로 인해 세밀한 디자인이나 작은 글자를 인식하기 어려움
  • 노화로 인해 섬세한 조작(예: 좁은 터치 인식 영역을 선택)에 피로감을 느낌

PROBLEMS

로그인 스크린

  • 메뉴들이 산개되어 요소들의 신속한 파악이 어려움
  • 로그인 버튼을 누르면 로그인 절차가 동일 화면에서 진행되지 않아 추가적인 행동을 필요로 하여 불편함을 유발

홈 스크린

  • 테스트 시작 버튼의 색상과 실제 테스트의 테마 색상이 서로 일치하지 않아 통일성을 해침
  • 다양한 디자인 요소로 인해 시선이 분산되어 ‘테스트 시작’의 즉각적인 실행이 어려움

메뉴 모음 스크린

  • 음성안내 버튼이 테스트 스크린 내부에 있지 않아 테스트 동안에 실시간으로 음성안내를 조작할 수 없어 불편함을 초래
  • 지난 점수 다시보기 페이지에서 가장 최근 점수만 표시되어 지난 테스트들의 결과 추이를 파악할 수 없음

테스트 시작 / 종료 스크린

  • 각 테스트 시작 / 종료 스크린이 텍스트로만 구성되어 해당 테스트 종류를 직관적으로 파악하기 어려움

테스트 스크린

  • 테스트 종류(치매, 우울증, 스트레스)마다 UI 디자인이 상이하여 각 테스트마다 새로운 UI 에 적응해야 하는 불편함을 초래
  • 질문 영역이 작고 시각적으로 구별이 크지 않아 질문의 즉각적인 파악이 어려움
  • 선택지의 터치 인식 범위가 좁아 세밀한 조작으로 인해 피로감을 유발
  • 그림 선택 항목에서 선택되지 않은 이미지의 투명도를 낮추는 방식이 이미지의 인식 및 선택을 어렵게 함

PROBLEM SOLVING

WIREFRAMES & WIREFLOW

  • 사용자가 UI를 신속하게 파악하고 행동을 취할 수 있도록 디테일한 디자인 요소를 배제하고 영역 위주로 뚜렷하게 구성
  • 다양한 기기 화면에서도 스크롤 없이(부수적 행동 최소화) 사용할 수 있도록 기기의 세로 길이에 비례하여 영역 크기 증감
  • 각 테스트에서 공통된 UI 디자인을 사용하여 새롭게 적응해야 하는 수고를 경감
  • 선택지를 버튼 형태의 UI로 교체하고 터치 인식 범위를 넓혀 보다 쾌적한 조작감을 제공

STYLE GUIDE : COLOR

  • 홈 스크린부터 테스트 전체, 지난 점수 다시보기까지 해당 테스트의 테마 컬러를 통일시켜 일관성을 유지
  • 주의가 필요한 상황(회원 탈퇴, 테스트 이탈 등)에는 대비되는 고채도의 적색을 활용하여 집중을 유도
  • 상호작용이 필수인 버튼 UI에 동일 색상을 반복적으로 사용하여 시각적으로 익숙함을 제공

STYLE GUIDE : FONT

  • 분명하고 신속한 텍스트의 의미 전달을 위해 고딕 계열의 글꼴을 채택
  • 사용자 연령층의 신체적 특성을 고려하여 통상적인 글꼴에서 크기를 확대하여 가독성을 확보
  • 질문에는 선택지보다 Bold 한 굵기를 사용, 시선을 끌어 사용자가 최우선적으로 인식할 수 있도록 유도
  • 40px 나눔바른고딕 Bold
  • 32px 나눔바른고딕 Regular
  • 24px 나눔바른고딕 Light
  • 16px 나눔바른고딕 UltraLight

SOLUTION

LOGIN SCREEN

로그인 과정을 한 화면 안에서 완료할 수 있도록 구성하였습니다. 부차적인 메뉴들은 화면 하단에 정렬하여 배치하였습니다.

HOME SCREEN

앱 로고 대신 텍스트를 삽입, 주목도를 높여 사용자의 즉각적인 테스트 실행을 유도하였습니다. 장식적인 디자인 요소를 지양하고 영역을 뚜렷하게 구분하여 각 요소들의 인식률을 높이는데 중점을 두었습니다. 더불어 하단의 CI 이미지를 ‘캐어유 치매 예방 게임 앱’ 마케팅 배너로 대체하여 테스트 이후 해당 앱 접근을 유도하였습니다.

TEST HISTORY SCREEN

최근 테스트 결과뿐만 아니라 지난 테스트 결과들을 둘러볼 수 있도록 재구성하였습니다. 스크롤 여부를 디자인 요소로 암시하는 대신 텍스트로 직접 명시하여 사용자에게 분명하게 행동을 제시합니다.

TEST START / FINISH SCREEN

테스트 시작과 완료 시에 텍스트와 더불어 테스트를 상징하는 아이콘을 추가하여 사용자가 해당 테스트 종류를 직관적으로 파악할 수 있도록 돕습니다.

TEST SCREEN : TEXT

상단에 위치한 질문을 사용자가 최우선적으로 인식하도록 배색과 Bold 한 글꼴을 활용하였습니다. 선택지는 터치 인식 가능 범위를 최대한 확보하는 데 중점을 두었습니다. 음성안내 UI가 아이콘 형태로 내부에 배치되어 사용자가 테스트 중 언제라도 접근할 수 있습니다.

TEST SCREEN : IMAGE

이미지 선택지는 투명도의 변화 대신 테두리와 함께 아이콘을 추가하여 모든 선택지에 대한 가시성을 확보하는 동시에 시각적인 대비 효과를 주었습니다.

TEST EXIT MODAL

테스트 도중 사용자의 조작 실수로 인한 이탈을 막고자 경고 문구와 아이콘을 포함한 모달(팝업) 창을 추가하였습니다.

TEST RESULT SCREEN

화면에서 가장 중요한 점수와 피드백 문구를 시각적으로 강조하여 주목도를 높였습니다. 해당 테스트의 소개와 판별 척도를 추가적으로 제공합니다.

ADS SCREEN

테스트 결과 이후 홈 스크린으로 이동 전 ‘치매 예방 게임 앱’을 노출시키는 마케팅 페이지를 추가하였습니다. 수익 창출을 위한 광고 영역으로의 확장을 고려하여 이미지 배너와 버튼 UI로 레이아웃을 구성하였습니다.

CONCLUSION
